# Hebrew Signs

**Hebrew letters are relational dynamics, not category labels.**

Each of the 22 Hebrew letters describes what a word *does* — how it connects, contains, reveals, or transforms. Three-letter roots are micro-sentences: the first letter initiates, the second mediates, the third completes. Reading roots this way illuminates actual word meaning **99.6% of the time** — tested across 500 randomly selected roots from the full Hebrew Bible, with every failure investigated.

This repository contains the evidence, tools, and data.

## Key Findings

1. **99.6% micro-sentence illumination.** 500 randomly selected three-letter roots were read as dynamic sequences (R1 initiates, R2 mediates, R3 completes). After investigating every failure: 21 were false positives (extraction noise, particles, conjugated forms), 12 resolved with polarity-aware reading, 18 resolved with concrete-meaning investigation. Only 2 roots remain genuinely weak. [Initial test](evidence/relational-synthesis.md) | [Failure investigation](evidence/phase3-failure-analysis.md)

2. **The 6Q Skeleton.** Six letters correspond to the six fundamental questions: Aleph=Who, Bet=What, Gimel=How, Dalet=Where, Qof=When, Tav=Why. The first four are sequential (positions 1-4). The last two appear near the end (positions 19, 22) — temporal and purposive questions require the earlier four. [Details](framework/6q-skeleton.md)

3. **Polarity-neutral dynamics.** The same letter dynamics produce devotion (Psalm 119), grief (Lamentations 1), and embodied virtue (Proverbs 31:10-31). The dynamics are structural, not emotional. Wickedness (rasha) reads as "leading that amplifies into distorted perception." Emptiness (riq) reads as "leading hand calling from depth with no response." The shape is identical; the valence depends on what fills it. [Lamentations test](evidence/lamentations-countertest.md) | [Proverbs 31 test](evidence/proverbs31-test.md) | [Polarity analysis](evidence/phase3-failure-analysis.md)

4. **Prefix letters prove the claim.** Hebrew's grammatical prefix letters already function as relational operators: Bet means "in/within" (containment), Lamed means "to/toward" (direction), Mem means "from" (extraction), Vav means "and" (conjunction). Our dynamics for these letters match their known grammatical function exactly. This isn't a coincidence — it's the principle that extends to all 22.

5. **Morphological architecture.** The triliteral root system functions as a compositional semantic architecture: 22 relational primitives (instruction set) + 3 positional roles (execution model) + 7 verb stems (function modifiers) + vowel patterns (parameters) + prefix letters (scope operators). Meaning is computed from structure, not arbitrarily assigned. [Full analysis](framework/morphological-architecture.md)

## Example: Genesis 1:1

```
bereshit         bara             elohim          et
In-beginning    created          God             (object marker)
Containment +   Containment ->   Origination ->  Origination ->
 Headship ->     Headship ->      Direction ->     Completion
 Origination ->  Origination      Revelation
 Intensification
```

**bereshit** (in the beginning): Containment (Bet) + the root rosh reads Headship -> Origination -> Intensification. "Within the leading-forth that establishes and amplifies."

**bara** (created): Containment -> Headship -> Origination. "Enclosing what leads toward establishing presence." Creation is giving form (containment) to what advances (headship) into being (origination).

**elohim** (God): Origination -> Direction -> Revelation. "That which establishes, directs toward, and makes visible."

**et** (direct object marker): Origination -> Completion. Aleph to Tav — first letter to last. The word that marks objects dynamically spans the entire alphabet.

## Try It Yourself

### Prerequisites

- Node.js 18+
- (Optional) An xAI API key for LLM-assisted meaning enrichment

### Quick Start

```bash
# Clone and enter the repo
git clone https://github.com/drapermcp/hebrew-signs.git
cd hebrew-signs

# Run the glosser on Genesis 1:1
node scripts/glosser.js "bereshit bara elohim"

# Run on a full chapter with traditional meanings
export XAI_API_KEY="your-key-here"  # Optional: for meaning enrichment
node scripts/glosser.js --chapter Gen.1 --enrich --format compact

# Run on a verse range
node scripts/glosser.js --verses Deu.6.4-9 --enrich

# Generate a formatted interlinear document
node scripts/interlinear.js --latest
```

### Getting the Full Corpus

The 305,000-word Tanach corpus is too large for this repository. To download and build it:

```bash
./scripts/download-corpus.sh
```

This clones the [UXLC Leningrad Codex](https://github.com/openscriptures/UXLC) XML and extracts the word corpus (~79MB).

## Methodology

1. **Corpus extraction.** Every Hebrew word from the Tanach (UXLC Leningrad Codex, 305,463 words, 39 books) extracted with consonantal decomposition.
2. **Letter statistics.** Frequency analysis, positional distribution, 22x22 co-occurrence matrix.
3. **Category-label scoring (Phase 3).** Top 75 words per letter analyzed by LLM for semantic tags. Average: 13.9%. Low — because it treats letters as categories.
4. **Relational-dynamic scoring (Phase 3b).** Same words re-analyzed for relational function. Average: 29.1%. Higher — because it asks what words DO.
5. **Root-position analysis (Phase 4).** Each letter examined as R1, R2, R3 in 4,617 extracted roots.
6. **Micro-sentence test (Phase 5).** 500 roots read as R1->R2->R3 dynamic sequences. Initial: 89% illumination. After failure investigation: **99.6%**.
7. **Failure analysis (Phase 3 revisited).** Every "weak" root investigated. 21 were methodology noise, 12 resolved by polarity awareness, 18 by concrete-meaning investigation. 2 remain genuinely weak.
8. **Acrostic validation.** Psalm 119, Lamentations 1, Proverbs 31:10-31 — three texts where each stanza begins with successive Hebrew letters. The dynamics match across devotion, grief, and virtue.
9. **6Q mapping.** Six letters identified as structural questions; sixteen as dynamic answers.
10. **Glosser and interlinear.** Automated tools producing dynamic readings for any Hebrew text.

Total computational cost: ~$1.00 in API calls. The entire evidence base runs in under two hours.

## Known Limitations

1. **2 roots don't illuminate.** ארצ (earth) and צנה (cold) remain genuinely weak. Earth's materiality and cold's sensation may lie outside the relational dynamics' reach.
2. **Verb stems (binyanim) not modeled.** Hebrew has 7 verb patterns that modify root meaning. The glosser handles base roots but doesn't yet capture causative/intensive/reflexive shifts.
3. **Root extraction is heuristic.** Prefix/suffix stripping approximates morphology but doesn't handle all patterns. Phase 3 showed this produces ~15% noise in root samples. A proper morphological parser would improve accuracy.
4. **LLM as evaluator.** We used Grok 4.1 to score alignments. Its assessments are informed but not authoritative. Human Hebrew scholars should verify key results.
5. **This is a layer, not a replacement.** The dynamic reading adds a dimension. It doesn't replace lexical, grammatical, or contextual translation.
